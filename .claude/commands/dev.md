# 開発オーケストレーター

このコマンドは開発フロー全体を統括するメインオーケストレーターです。

## 使用方法

```
/dev [タスクの説明]
```

## 入力パラメータ
$ARGUMENTS

## タスク開始

以下の手順で開発を進めます：

### Step 1: 要件分析
まず、タスクを分析し、必要なサブエージェントを特定します。

### Step 2: 設計フェーズ
デザインが必要な場合：
- `/design` でページ設計を行う
- `/image-gen` で必要な画像を準備

### Step 3: 実装フェーズ
- バックエンド変更が必要な場合：`/backend`
- フロントエンド実装：`/coding`

### Step 4: テストフェーズ
- `/debug` でテストを実行
- バグがあれば `/coding` で修正

### Step 5: 評価フェーズ
- `/persona` でペルソナ評価（40点以上必須）
- `/evaluate` でSEO・事業評価（70点以上必須）

### Step 6: 反復
評価で不合格の場合：
1. 改善点をリストアップ
2. 該当するサブエージェントで修正
3. 再度 `/debug` → `/persona` → `/evaluate`

## サブエージェント一覧

| コマンド | 役割 | 出力 |
|---------|------|------|
| `/design` | ページデザイン | ワイヤーフレーム、カラースキーム |
| `/image-gen` | 画像プロンプト作成 | 画像生成用プロンプト |
| `/coding` | フロントエンド実装 | HTML/CSS/JS |
| `/backend` | バックエンド実装 | サーバーサイドコード |
| `/debug` | テスト・デバッグ | バグレポート |
| `/persona` | ペルソナ評価 | ユーザー視点スコア |
| `/evaluate` | 事業評価 | SEO・CV最適化スコア |

## 品質基準

### 合格条件
1. **デバッグ**: 全テスト合格
2. **ペルソナ評価**: 各ペルソナ平均40点以上（50点満点）
3. **事業評価**: 70点以上（100点満点）

### デザイン原則
- 紫色・AIっぽいデザインは禁止
- 絵文字の多用は禁止
- 温かみのある色合いを使用
- アクセシビリティを重視

**開始するには、タスクの説明を入力してください。**

例：
- `/dev 新しいランディングページを作成`
- `/dev お問い合わせフォームを追加`
- `/dev ダッシュボードのデザインを改善`
