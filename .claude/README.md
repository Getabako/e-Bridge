# e-Bridge サブエージェントシステム

このディレクトリには、e-Bridge開発用のサブエージェントシステムが含まれています。

## 概要

各サブエージェントは特定の役割に特化しており、スラッシュコマンドで呼び出せます。
開発タスクはエージェント間で引き継がれ、最終的にペルソナ評価・事業評価を通過することで完了となります。

## クイックスタート

```bash
# 開発フロー全体を開始
/dev [タスクの説明]

# 例
/dev 新しいランディングページを作成
/dev お問い合わせフォームを追加
/dev ダッシュボードのUIを改善
```

## サブエージェント一覧

| コマンド | エージェント名 | 役割 |
|---------|--------------|------|
| `/dev` | オーケストレーター | 開発フロー全体を統括 |
| `/design` | デザインエージェント | ページデザイン・UI/UX設計 |
| `/image-gen` | 画像生成エージェント | 画像プロンプト作成 |
| `/coding` | コーディングエージェント | フロントエンド実装 |
| `/backend` | バックエンドエージェント | サーバーサイド実装 |
| `/debug` | デバッグエージェント | テスト・品質保証 |
| `/persona` | ペルソナ評価エージェント | ユーザー視点での評価 |
| `/evaluate` | 事業評価エージェント | SEO・事業観点での評価 |

## 合格基準

| 評価 | 基準 |
|------|------|
| デバッグ | 全テスト合格 |
| ペルソナ評価 | 各ペルソナ平均40点以上（50点満点） |
| 事業評価 | 70点以上（100点満点） |

## ペルソナ

評価に使用する3つのペルソナ:

1. **ユウキ（中2・男子）** - 吃音があり、人前で話すことに不安
2. **サクラ（高1・女子）** - 不登校経験あり、ADHD、通信制高校
3. **タクミ親子（小5・男子）** - ASD傾向、感覚過敏、母親も評価対象

詳細は `.claude/skills/personas.md` を参照

## デザインルール

### 禁止事項
- 紫色・パープル系の使用
- 絵文字の多用
- AIっぽいグラデーション
- ネオン・サイバーパンク風デザイン

### 推奨カラー
- メイン: `#2E7D32`（フォレストグリーン）
- アクセント: `#FF8F00`（ウォームオレンジ）

詳細は `.claude/skills/brand-guidelines.md` を参照

## 使用例

### 例1: ランディングページの作成

```bash
# 開発開始
/dev ランディングページを作成

# デザインフェーズ
/design ヒーローセクション、サービス紹介、CTAを含むランディングページ

# 画像準備
/image-gen ヒーロー画像とサービスアイコン

# 実装
/coding デザイン仕様に基づいてHTML/CSS/JSを実装

# テスト
/debug 機能テストとアクセシビリティテスト

# 評価（合格するまで繰り返し）
/persona
/evaluate
```

### 例2: 既存ページの改善

```bash
# 問題箇所のデバッグ
/debug ダッシュボードのパフォーマンス問題を調査

# 改善実装
/coding パフォーマンス改善

# 再評価
/persona
/evaluate
```

## トラブルシューティング

### 評価で不合格になった場合
1. 評価レポートの改善提案を確認
2. 該当するエージェント（/design, /coding等）で修正
3. /debug で再テスト
4. /persona → /evaluate で再評価

### エージェント間の引き継ぎ
各エージェントのレポート末尾に「次のエージェントへ」というセクションがあります。
その指示に従って次のコマンドを実行してください。
